The Masterclass Corrections
Here is exactly where you lost points. Update your mental model with these two concepts:

1. Task 3: The S3 Syntax Trap (Score: 0/1)
You missed the biggest trap in AWS Data Engineering! In your answer, you wrote: "resource": "arn/.../fincial-data".

If you deploy that code, your script will crash with an "Access Denied" error the second it tries to download a file.
AWS treats the Bucket and the Files inside the bucket as two completely different things.

To look at the bucket itself (s3:ListBucket), you need the bucket ARN.

To touch the files (s3:GetObject), you need the bucket ARN plus /*.

The correct JSON array looks exactly like this:

JSON
"Resource": [
  "arn:aws:s3:::finance-data",
  "arn:aws:s3:::finance-data/*"
]
(Also, remember that IAM policies are strictly JSON, so avoid writing them in YAML format with missing brackets and commas!)

2. Task 6: SCP vs RCP (Score: 0.5/1)
You wrote: "RCP states who can access and interact with a resource and what actions they can perform..."

You fell into the "Granting" trap.
Neither an SCP nor an RCP can state who can access something. They never grant access. They only act as a ceiling/maximum limit.

SCP: Sets a maximum boundary on the Identities (Users/Roles) in an account. (e.g., "No user in this account can delete databases.")

RCP: Sets a maximum boundary on the Resources (S3 buckets, etc.) in an account. (e.g., "No S3 bucket in this account can ever be exposed to the public internet.")
